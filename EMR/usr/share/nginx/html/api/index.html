ISEmr API docs

ID

To give the API a patient id just add id as query string.

Example

/api/get_allergies.php?id=3

End-points

/api/get_allergies.php

GET Get the allergies for the patient id

/api/get_appointments.php

GET Get the appointments for the patient id

/api/get_medical_issues.php

GET Get the medical issues for the patient id

/api/get_medications.php

Get the medications for the patient id

/api/get_medications.php

Get the patient data

GET Get the medications for the patient id

/api/get_patient_data.php

GET Get the patient data

/api/create_patient.php

POST In order to create a patient post a JSON object with the following attributes. Attributes marked with a "*" are required. All others are not needed.

  • title
  • language
  • financial
  • fname*
  • lname*
  • mname
  • DOB (YYYY-MM-DD)*
  • street
  • postal_code
  • city
  • state
  • country_code
  • drivers_license
  • ss (SSN in the format of: NNN-NN-NNNN)
  • occupation
  • phone_home
  • phone_biz
  • phone_contact
  • phone_cell
  • pharmacy_id
  • status (marital)
    • null (Unassigned)
    • married
    • single/option>
    • divorced
    • widowed
    • separated
    • domestic Partner
  • contact relationship
  • sex*
    • Female
    • Male

/api/create_allergy.php

In order to create an allergy for a patient post a JSON object with the following attributes. Attributes marked with a "*" are required. All others are not needed.

  • pid* (The id of the patient who has the allergy)
  • title* (title of the allergy)
  • Reaction
  • Comments
  • Occurrence
    • 1 (First)
    • 6 - (Early Recurrence (<2 Mo))
    • 7 - (Late Recurrence (2-12 Mo))
    • 8 - (Delayed Recurrence (> 12 Mo))
    • 4 - (Chronic/Recurrent)
    • 5 - Acute on Chronic
  • Severity
    • Mild
    • Mild to moderate
    • Moderate
    • Moderate to severe
    • Severe
    • Life threatening severity
    • Fatal
  • Outcome
    • 0 - (Unassigned)
    • 1 - (Resolved)
    • 2 - (Improved)
    • 3 - (Status quo)
    • 4 - (Worse)
    • 5 - (Pending followup)

api/set_credentials.php

POST In order to create an medication for a patient post a JSON object with the following attributes. Attributes marked with a "*" are required. All others are not needed.

  • SSN (Should be in the format NNN-NN-NNNN)
  • Username* (This will be the new username)
  • Password* (The Password you like to set for the user)
